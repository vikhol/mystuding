Глава 4

Использование HTML5 в целях разработки адаптивного веб-дизайна.

  • lang="en" указывает основной язык содержимого элементов и любого из атрибутов элемента, содержащего текст. Если вы не пишете страницы на английском, то лучше указать правильный код языка. Для русского указывается ru.

  • В <a> можно заключать целые наборы других тегов. Например, <h1> + <p> + <img>. Но есть и иключения: другой тег <a>, <button>, или какие либо формы.

  • <main> - представляет собой блок основного содержимого страницы, включает уникальную для этой страницы информацию и исключает содержимое, повторяющееся в наборе документов, например, навигационные ссылки, информация об авторском праве, логотипы и баннеры сайта и формы для поиска данных (если только основной функцией документа или приложения не является именно поиск данных)».
  • На странице не может быть несколько блоков <main>. А так же <main> не может быть "ребенком" относительно article, aside, header, footer, nav и header, но вполне может являться их родителем.

  • <section> подходит для таких элементов страницы, как элементы списка новостей, элементы списка товаров в каталоге, переключающихся табов. Т.е. для подобных друг-другу элементов и блоков в одном родителе и имеющих заголовки. Так же section может использоваться для разбиения текста на части. Например главы. Использование section как аналога article - неправильно и противоречит спецификации. (Т.е. проще говоря section это аналог тега li только для блоков страницы).

  • <article> - контейнер для законченных фрагментов содержимого.

    • Пример разницы:

    <article> - статья

    <section> - кусок этой статьи или любой кусок некоторого целого материала.

  • <nav> используется в качестве контейнера основных навигационных ссылок на другие страницы или фрагменты внутри той же самой страницы.

  • <aside> - контейнер, который можно использовать в качесте сайдбара, блоков рекламы, цитат, или блока «клиенты, купившие этот товар, приобрели также...»

  • <figure> используется для группирования любых элементов, например, изображений и подписей к ним.
  • <figcaption> пригодится тогда, когда изображениям или коду нужны небольшие сопроводительные подписи, которые не будут соответствовать основному тексту содержимого.

  • <details> и <summary> (при нажатии на эту "ссылку" откроется описание)

    Является чем-то вроде spoiler'а. При этом <details> - используется для хранения информации, которую можно скрыть или показать по требованию пользователя. <summary> заголовок для тега <details>, по которому можно щелкать для разворачивания/сворачивания информации. Не поддерживает (без использования JS) анимацию. Не поддерживается в IE и Opera Mini. Пример естественно в коде этой страницы.

    Чтобы сменить или убрать стандартную иконку нужно прописать:

    summary::-webkit-details-marker {

    display: none;

    }

  • <header> используется как хедер основной страницы, но не только. Его можно использовать не ограниченное кол-во раз на странице, например, внутри каждого элемента <section>

  • <footer> так же можно использовать, как для "подвала" основной страницы, так и в качестве подвальной части отдельной публикации.

  • <address> предназначен для хранения информации об авторе и может включать в себя любые элементы HTML вроде ссылок, текста, выделений и т.д. Планируется, что поисковые системы будут анализировать содержимое этого тега для сбора информации об авторах сайтов.
