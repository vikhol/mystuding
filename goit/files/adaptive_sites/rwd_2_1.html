Глава 2

Медиазапросы.

Пример 1 - изменение цвета бэкграунда в зависимости от ширины видимой области.

  • В HTML (в теге head) можно указать, какие файлы стилей будут использоваться, в зависимости от прописанных медиазапросов.
    • </head>
    <link rel="stylesheet" media="(min-width: 300px)" href="w300.css" />
    <link rel="stylesheet" media="(min-width: 600px)" href="w600.css" />
    </head>
  • Если при загрузке страницы, медиазапрос в head не подходит под текущую ситуацию, то соответствующий CSS-файл обработается позже. Это помогает в повышении производительности загрузки страниц. Но прирост небольшой и им можно пренебречь. Пример

  • Лучше таки встраивать медиазопросы в общий CSS-файл.

  • @media - указывает на необходимость медиазапроса. Затем:
    1. задается тип с которым требуется совпадение. Например: screen (не обязательно)
    2. Затем внутри скобок вводятся особенности запроса. Например: (max-device-width: 400px)
    3. И далее открываются фигурнуе скобки, внутри которых уже следуют необходимые стили.

  • В большинстве случаев, указывать screen не нужно. Если тип медиаустройства не задан конкретным образом, подразумевается настройка all.
  • В медиазапрос может быть заключен любой код CSS.

    • Cтили, находящиеся ниже в CSS-файле, доминируют над такими же стилями, находящимися выше (если только расположенные выше стили не имеют более конкретной нацеленности).

  • Поэтому в начале файла CSS можно прописать общие стили, которые будут применяться ко всем версиям сайта, а затем предопределить стили с помощью медиазапросов.

    • С помощью медиазапросов можно задать правила стилей, которые будут использоваться при наличии, или отсутствии у пользователя включенного JavaScript.

    1. @media (scripting: none) {

      /* стили, предназначенные для применения в отсутствие JavaScript */

      }

    2. @media (scripting: enabled) {

      /* стили, предназначенные для применения при наличии JavaScript */

      }

    viewport

  • viewport добавляется в head.

  • Он может настраиваться на определенную ширину, или содержать указание на масштаб.

    • <meta name="viewport" content="initial-scale=2.0, width=device-width" />

    1. content="initial-scale=2.0 - предписывает увеличить размер вдвое (от исходного)
    2. width=device-width - говорит браузеру, что ширина страницы должна быть равна значению device-width.

    В большинстве случаев подойдет следующий метатег:

    <meta name="viewport" content="initial-scale=1.0, width=device-width" />
