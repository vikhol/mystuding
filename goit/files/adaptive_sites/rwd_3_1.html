Глава 3

Динамически изменяемые разметки и адаптивные изображения.

Преобразование дизайна с фиксированными размерами в подстраеваемый.

Общая формула:
Цель / Среда = Результат.

Пример:

Имеется страница с фиксированными размерами: 200px - левая колонка + 660px - основная центральная + 100рх - правая колонка = 960рх стандартный размер страницы.

Чтобы преобразовать эти размеры в подстраиваемые и пропорциональные, нужно воспользоваться формулой выше.

  • Сперва берем левую колонку и подставляем в формулу: 200 / 960 = 0,208333333
  • Далее, в этом конкретном случае, чтобы расчитать процентное соотношение 200-от пикселов к 960-ти, нужно перенести запятую на 2 позиции вправо. В итоге получаем 20,8333333 %
  • Так же поступаем со средней и правой колонками. Получаем 68,75 % и 10,4166667 % соответственно.
    • Результат

    Такое строрение страницы позволит всем эл-там оставаться пропорциональными друг к другу. Но при слишком мелких размерах, некоторые колонки становятся слишком узкими. Поэтому нужно добавить контрольную точку, которая в какой-то момент выстроит все эл-ты в одну колонку.

    Flexbox

    Кратко о свойствах:

  • Он легко справляется с выравниванием содержимого по вертикали.
  • Может изменить порядок визуального отображения эл-тов.
  • Может авт-ски заполнять пространство пробелами и выравнивать эл-ты внутри блока, автоматически распределяя между ними доступное пустое пространство.

    • У Flexbox имеются 4 основных параметра:

    1. direction - направление.
    2. alignment - выравнивание.
    3. ordering - определение порядка.
    4. flexibility - динамическое изменение.

    Текст безупречно выровненный по вертикали.

    Пример 03-03.

    В CSS-файле примера нам интересны следующие св-ва:
  • display: flex; - настройка элемента на его принадлежность к Flexbox.
  • align-items: center; - выравнивает эл-ты по поперечной оси.
  • justify-content: center;- задает выравнивание содержимого по центру главной оси.

    • Пример 03-04 - в этом примере особую роль играет св-во margin-left: auto;, которое применяется к последнему элементу и заставляет его отступить вправо на максимально доступную ширину.

    Св-ва выравнивания для FlexBox

    Пример.
  • Сперва важно разобраться с концепцией осей: главная (main axis) и поперечная (cross axis).
  • Например, если направление Flexbox-контейнера настроено на строку, главная ось будет горизонтальной, а поперечная ось — вертикальной. И наоборот, если направление настроено на столбец.

    • Выравнивание по поперечной оси.

  • align-items позиционирует элементы относительно поперечной оси.
  • align-self используется для выравнивания "себя" отдельных flex-элементов. Может иметь следующие значения:
    1. flex-start - заставит эл-т начинаться с начальной границы своего flex-контейнера.
    2. flex-end - заставит эл-т выровняться по конечной границе своего flex-контейнера.
    3. center - заставит элемент расположиться посредине flex-контейнера.
    4. baseline - заставит все flex-элементы в контейнере выровняться по нижним строкам.
    5. stretch - заставит элементы растянуться по размеру их flex-контейнера (по поперечной оси).

    • Выравнивание по главной оси.

  • justify-content - отвечает за выравнивание по главной оси (justify-self для эл-тов, не находящихся в flex-контейнере.). Может иметь следующие значения:
    1. flex-start (аналогично таким же св-вам для align-items)
    2. flex-end (аналогично таким же св-вам для align-items)
    3. center (аналогично таким же св-вам для align-items)
    4. space-between - заставит распределить свободное пространство поровну между элементами.
    5. space-around - распределит свободное пространство в равных долях вокруг элементов.

    • Свойство flex.

    Cв-во flex является сокращенным определением трех отдельных свойств: flex-grow, flex-shrink и flex-basis.
  • flex-grow - является величиной относительно других flex-элементов, до которой может увеличиваться flex-элемент при наличии свободного пространства.
  • flex-shrink является величиной, до которой может уменьшаться flex-элемент относительно других flex-элементов, когда имеется дефицит пространства.
  • flex-basis является базовым размером, к которому приводится размер flex-элемента.
    • flex: 0 0 50px; означает, что элемент не будет ни увеличиваться, ни уменьшаться и его базовая величина составит 50px (то есть он будет иметь размер 50 пикселов независимо от величины свободного пространства)
  • Если установить для свойства flex-shrink нулевое значение, то свойство flex-basis поведет себя как задание минимальной ширины.
  • flex можно представлять в виде способа задания отношений. Когда для каждого flex-элемента этому свойству присвоено значение 1, то все они занимают одинаковое пространство.

    Изменение порядка следования эл-тов

    Пример

    order - позволяет пересмотреть порядок следования элементов внутри Flexbox.

  • order: -2; будет стоять на позицию выше в списке, чем order: -1;
  • По умолчанию имеет значение order: 0;
  • flex-flow: row wrap; позволяет flex-элементам располагаться на нескольких строках. Данное свойство относится к числу наименее поддерживаемых, поэтому в зависимости от того, насколько глубоко в прошлое должна простираться поддержка вашего продукта, может понадобиться заключить содержимое и две боковые панели в другой элемент

    • Адаптивные изображения.

    Наиболее распространенные сценарии, для которых понадобятся адаптивные изоб-я:

    1. Они (сценарии) будут переключать изображение, когда потребуется другое разрешение.
    2. Полностью менять изображение в зависимости от доступного пространства окна просмотра.

    srcset

    Пример:
    <img src="scones_small.jpg" srcset="scones_medium.jpg 1.5x, scones_large.jpg 2x" alt="Scones taste amazing">
    1. Сначала идет src и указывает на дефолтную версию изобр-я. (Которая так же будет использоваться, если браузер не поддерживает srcset)
    2. Далее через запятые идет список изображений из которых браузер сделает выбор.

    picture (не поддерживается в IE и Opera mini) - контейнер, который содержит в себе все варианты изображений и может выводить нужное из них, в зависимости от заданного медиа запроса.

    Пример использования:

    <picture>

    <source media="(min-width: 30em)" srcset="cake-table.jpg">

    <source media="(min-width: 60em)" srcset="cake-shop.jpg">

    <img src="scones.jpg" alt="One way or another, you WILL get cake.">

    </picture>

    Тег img в данном контейнере обязателен!

  • picture можно использовать для отображения изображений в формате WebP (если он поддерживается браузером).
    Пример:

    <picture>

    <source type="image/webp" srcset="scones-baby-yeah.webp">

    <img src="scones-baby-yeah.jpg" alt="Again, you WILL eat cake.">

    </picture>

    type="image/webp" работает примерно так же, как и медиазапрос в предыдущем примере.
