Типы, равенства, преобразования и т.д.

(types.js)

Типы операндов бывают следующих видов: “string" - строка, “boolean" - логический тип (булевское значение), “number" - число, “object" - объект, “undefined" - неопределенный, “function" - функция, и т. д.

Для проверки типа операнда используется встроенный оператор typeof. (Пример в types.js)

Примитивы: числа, строки, булевские значения, null и undefined. Все остальное — объекты.

undefined означает, что переменная (или свойство, или элемент массива) еще не была инициализирована значением. Например:

  1. Возвращает ф-я при отсутствии в ней команды return.
  2. Является значением переменной, которой еще не присвоили значение.
  3. Значение эл-та не существующего в разреженном массиве.
  4. Значение несуществующего св-ва.
  5. и т.д.
  • Поэтому undefined может использоваться для проверки заполнения форм, объектов и т.д.

    • null обозначает отсутствие объекта.

  • Может использоваться в коде, например так: var header = document.getElementById("header"); if (header == null) {...}

    • NaN (Not a Number) - используется для представления числовых рез-тов, не имеющих представления. Например:

  • var x = "stroka" + 99; - не сможет сложить слово с числом и вернет NaN.
  • А так как NaN не равно ничему (даже самому себе), его не получится проверить стандартными способами, как в случае с null и undefined. Для этого используется специальная ф-я isNan.
  • Например: if (isNaN(moeChislo)) {...}; - Функция isNaN возвращает true, если переданное ей значение не является числом.

    • Равенства и преобразования.

    == - оператор проверки равенства.

  • Если два знач-я относятся к одному типу, то == просто сравнивает их.
  • А если знач-я относятся к разным типам, JavaScript сперва пытается преобразовать их, и уже потом сравнить. Т.е. когда мы пытаемся сравнить строку с числом, JS пробует преобразовать строку в число.
  • Если преобразуемая строка не содержит число, то и преоразовать ее получится только в NaN. Пример: 99 == "строка" -> 99 == NaN -> false.
  • В случае сравнения булевского значения с числом, true преобразуется в число 1, а false в 0.
  • Так же "1" == true. Сперва true преобразуется в число. А затем строка "1" сравнивается с числом 1. T.e. "1" == true; -> "1" == 1; -> 1 == 1; true
  • null == undefined -> true
  • 1 == "" -> 1 == 0 -> false

    • === - оператор "Строгое равенство", или оператор тождественности

  • При использовании ===, два значения считаются строго равными только в том случае, если они относятся к одному типу и содержат одно значение.
  • !== - строгое неравенство

    • Одна строка может быть меньше или больше другой. Например: "banana" < "melone" Потому что в алфавите первая буква слова banana предшествует первой букве melone.

    О канкатенации (склеивании строк) и суммировании

    Оператор + суммирует числа, а строки склеивает.
  • var x = "4" + 3 -> x = 43
  • Но поскольку оператор + обладает так называемой левосторонней ассоциативностью, в следующей ситуации: var order = 1 + 2 + " pizzas"; мы получим строку “3 pizzas”, а не "12 pizzas"
  • Чтобы JS преобразовал строку в число и сложил его с другим числом, нужно использовать встроенную ф-ю Number. П-р: var x = 3 + Number("4"); -> x = 7.

    • Другие арифметические операторы ("-" вычитание, "/" деление, "*" умножение) будут в этом случае преобразовывать строку в число и работать как с числом.

  • var x = "4" * 3 -> x = 12
  • var x = 10 / "2" -> x = 5
  • var x = 4 - "3" -> x = 1

    • Как проверить два объекта на равенство

  • При проверке равенства двух объектных переменных сравниваются ссылки на эти объекты.
  • Две ссылки равны только в том случае, если они ссылаются на один объект.
